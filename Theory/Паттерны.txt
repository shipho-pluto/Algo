                                                                                    1. Порождающие паттерны (Creational)


Отвечают за создание объектов, скрывая детали инстанцирования.

Singleton (Одиночка)
Гарантирует, что у класса есть только один экземпляр.
→ Пример: глобальный конфиг, логгер.

Factory Method (Фабричный метод)
Определяет интерфейс для создания объекта, но оставляет подклассам решение о том, какой класс инстанцировать.
→ Пример: создание документов в редакторе (Word/PDF).

Abstract Factory (Абстрактная фабрика)
Создание семейств связанных объектов без указания их конкретных классов.
→ Пример: GUI-библиотеки (кнопки/окна для Windows/macOS).

Builder (Строитель)
Позволяет создавать сложные объекты пошагово.
→ Пример: конструктор SQL-запросов.

Prototype (Прототип)
Копирование существующих объектов без зависимости от их классов.
→ Пример: клонирование игровых персонажей.

                                                                                    2. Структурные паттерны (Structural)


Организуют классы и объекты в более крупные структуры.

Adapter (Адаптер)
Преобразует интерфейс класса в другой интерфейс, ожидаемый клиентом.
→ Пример: адаптер для работы со старым API.

Decorator (Декоратор)
Динамически добавляет объекту новые обязанности.
→ Пример: добавление функционала к файловому вводу/выводу (сжатие, шифрование).

Facade (Фасад)
Простой интерфейс для сложной подсистемы.
→ Пример: упрощённый API для работы с базой данных.

Composite (Компоновщик)
Объединяет объекты в древовидные структуры.
→ Пример: GUI (контейнеры + кнопки/текст).

Proxy (Заместитель)
Контролирует доступ к объекту, добавляя дополнительную логику.
→ Пример: ленивая загрузка (Lazy Load), кеширование.

                                                                                  3. Поведенческие паттерны (Behavioral)


Решают задачи эффективного взаимодействия объектов.

Observer (Наблюдатель)
Уведомляет зависимые объекты об изменениях состояния.
→ Пример: система событий в UI (кнопка → обработчики).

Strategy (Стратегия)
Инкапсулирует алгоритмы в отдельные классы, делая их взаимозаменяемыми.
→ Пример: сортировка (QuickSort vs MergeSort).

Command (Команда)
Инкапсулирует запрос как объект.
→ Пример: очередь операций «Отменить/Повторить».

Iterator (Итератор)
Последовательный доступ к элементам коллекции без раскрытия её структуры.
→ Пример: обход элементов списка, дерева.

State (Состояние)
Позволяет объекту менять поведение при изменении состояния.
→ Пример: автомат состояний (лифт: движение/двери открыты).
